⚡ QUICK 3-STEP DEPLOYMENT
═══════════════════════════════════════════════════════════════════════════════

Step 1: Generate Configuration (2 minutes)
  $ python generate_deployment_config.py
  
  This will:
    • Generate your Django SECRET_KEY
    • Create .env template (optional)
    • Show all needed environment variables

Step 2: Setup Database (3 minutes)
  
  Go to: https://neon.tech
  • Create PostgreSQL database
  • Save credentials (host, user, password)

Step 3: Deploy to Render (5 minutes)
  
  Go to: https://dashboard.render.com
  1. Click "New +" → "Web Service"
  2. Connect GitHub: joji309/St-thomas-church
  3. Add environment variables (from Step 1)
  4. Click "Create Web Service"
  5. Done! Deployment starts automatically

⚙️ ENVIRONMENT VARIABLES NEEDED
═══════════════════════════════════════════════════════════════════════════════

Core Settings:
  DEBUG                 = False
  SECRET_KEY            = (run generator script)
  ALLOWED_HOSTS         = your-render-url.onrender.com

Database:
  DB_ENGINE             = django.db.backends.postgresql
  DB_NAME               = neondb
  DB_USER               = neondb_owner
  DB_PASSWORD           = (from database)
  DB_HOST               = (from database)
  DB_PORT               = 5432

CORS:
  CORS_ALLOWED_ORIGINS  = https://your-render-url.onrender.com

👉 Run generator script to see all variables:
   $ python generate_deployment_config.py

✅ Environment Variables:
   Set them in Render dashboard → NEVER in .env file
   
✅ Debugging:
   Watch logs in Render dashboard real-time during deployment
   
✅ Auto-Deploy:
   Future git pushes to 'main' auto-deploy (no action needed)


🆘 WHEN THINGS GO WRONG
═══════════════════════════════════════════════════════════════════════════════

1. Check Render Logs:
   → Most detailed source of errors
   → Render dashboard → Logs tab

2. Review Documentation:
   → TROUBLESHOOTING.md has common fixes
   → RENDER_DEPLOYMENT_CHECKLIST.md for verification

3. Common Issues:
   → Build error? Check requirements.txt
   → 404 error? Update ALLOWED_HOSTS
   → CSS missing? Check collectstatic in logs
   → DB error? Verify credentials in env vars
